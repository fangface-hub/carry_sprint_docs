# CarrySprint Terms

## CarrySprint

A budget-first sprint planning and carry-over support tool.

## Budget-in

The set of tasks that fit within the available work-hour budget for the current sprint.

## Budget-out

The set of tasks that do not fit within the available work-hour budget and remain outside the current sprint scope.

## Carry-over

The act of deferring a task to the next sprint without deleting it.

## Impact

A relative expression of task value used to support prioritization.

## Estimated hours

The forecast work-hours required to complete one task.

## Productivity coefficient

A calculation factor used to adjust available work-hours for sprint planning.

## Project storage

The SQLite file that stores one project's data.

## Sprint workspace

The main screen where users review tasks, compare budget-in and budget-out items, and adjust decisions.
